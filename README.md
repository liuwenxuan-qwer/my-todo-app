## 3. 需求理解以及需求实现
**需求理解+实现**
添加待办事项（包含标题，描述可选）。
删除待办事项。
标记待办事项完成/未完成。
查看待办事项列表。
任务排序（设置任务时有一个优先级，优先级不同颜色深度不同）
任务分类（设计几种任务分类 工作，社交，家庭，学习等有不同的颜色）
提醒通知功能（今日代办展示，代办前一天 弹出提醒框）
行程分享功能，加入一个组织，可以看到别人的行程安排（单独页面）。
常规的登入功能以及邮箱注册等

**需求实现和页面设计**

一、导航页面结构
导航栏 包括 主页面 行程分析页面 登录按钮（未登录状态）或个人中心按钮（登录后状态）

二、主页面 由一个大型日历加上一个代办事宜组成，左右分布日历为举行，右边为代办事项展示（默认是今日），当点击日历中的日期时，跳转到当日的代办事项。
代办事框添加功能：
添加待办事项（包含标题，弹窗添加）。
删除待办事项（每个代办事宜有一个小垃圾箱图标）
标记待办事项完成/未完成。（垃圾箱图标前有一个完成和未完成的图标，支持点击切换状态）

三、行程分享功能页面：
页面分为三个部分，
1.左半是组织管理展示页面，包括你当前参加的组织，
组织快速切换区（固定高度约200px）
显示当前用户加入的所有组织（同顶部下拉）
当前选中组织高亮加粗 + 小勾
最下方永远显示加入组织，创建组织按钮
加入组织弹窗设计，通过邀请码加入。
创建组织也是弹窗设计，设置邀请码，以及组织信息。
2.中间部分：当前组织成员列表（可滚动）
只有切换到具体组织后才显示
选择“个人日历”时此区域自动隐藏或显示“个人模式”
成员排序：管理员在前 → 在线在前 → 姓名排序
每个成员右侧显示“今日公开行程数量”小角标
支持多选 + “只看选中的成员”开关（实时过滤右侧日历）
3.右侧主日历区（修改）
默认周视图（最适合团队）
支持今天/周/月 三种视图切换
通过列表项的方式展示，周和月就在上分添加一个日期选择器，然后下方跟着一个列表项（这个列表项可设计信息紧凑一点高度不要太高，因为人员可能会比较多）
在这里可以看到其他人员的行程安排，主要是工作和社交类的，像家庭学习类的不共享

登录后，用户可进入“个人中心”，
 展示用户基本资料
